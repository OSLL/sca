[up]Overview

Source Code Analyzer (SCA) is a tool to maintain knowledge base around the sources and projects.

SCA has several workplaces you can interact with:

  1. File browser
  2. Text view
  3. Binary view
  4. Graph view
  5. Property browser

[up]Workplaces

[up]File browser

You can use file browser as your usual tree-browser: expand folders, fold them etc.

To use file you can use context menu or just drag'n'drop file to view which you want to use.

File browser also supports annotating directories and files without creating objects. Simply call context menu on file/directory and select "Edit annotation".

NOTE: As a default, text view opens file using UTF-8 encoding, but you can force it to open in different encoding by calling context menu on file, selecting “Open in Text viewer as” and then selecting your preferable encoding out of supported ones.

[up]Text view

Text view provides viewing files in different encodings: UTF-8, CP-866, ISO-8859-15.

It also supports drag’n’drop: just select any text and drag it to scene.

[up]Binary view

Binary view provides viewing files as a HEX-files as any usual hex-editor. On the left you can see current offset of line in the table, in the next column - bytes loaded and separated by spaces, on the right you can see their symbolic representation.

To use binary view's drag'n'drop select bytes you want and drag them to graph view.

[up]Graph view

Graph view is a workplace where you can drag’n’drop. object to.

To select one object click on it, so it become highlighted. If you want to select multiple objects, you can hold Ctrl and invert selection of objects by clicking on them. Also, you can draw a rectangle to select multiple object. To do so, hold your left mouse button starting on any free place on graph, move your mouse (see the rectange on graph which selects objects), release the button when desired selection reached. Clicking on free place resets selection.

You can call context menu of objects by right-clicking on them. Menu's entries are: connect (creates link between objects), delete, converting entries, link's arrows entries, edit annotation.

To see information about object click on it and use property browser, or hold your mouse on object to see simple tooltip.

[up]Table view

Table view provides another view of objects, which is preferrably made to be used to go to filtered objects by double-clicking on objects in graph. If filter is empty or nothing matches filter, table view shows all object of graph.

[up]Property browser

Property browser provides seeing some fields of objects on graph.

We do not recommend to edit objects' path, but you can do it (e.g. you renamed file object linked to and want to fix double-click moving to object).

[up]Using external tools

While observing source, you may need to see what other tools make out of your objects. To use them, simply follow to the list of tools in menu Edit->Settings->Tools, then click add and follow hints of syntax. For the beginning, we added some tools for you by default, but if you wish, you can remove them.

After making list of tools you wish to use, click OK. Now to call them right click on folders/files at file browser or directly on objects of graph view. Window with output of command appears and you can operate with that like with a text in text view.

[up]Annotating

You can annotate any object on graph by right-clicking on it and selecting "Edit annotation" entry in menu. Annotation representation on graph supports HTML formatting.

See also annotating directory/file in file browser.

[up]Object types

Directory/File

Simple directory of file dragged from file browser. Double click scrolls to it in file browser.

Text block

Text dragged from text view. Double click scrolls to file in file browser + opens file in both text and binary views and selects it.

Identifier

Pretty much the same as text block, just has another default representation. Double click does same as text block.

Line

Same as text block, but contains the whole line of characters in file. Double click does same as text block.

Symbol

Simpler version of text block, which contains only one specific character. Double click does same as text block.

Binary block

Block of bytes dragged from binary view. Double click does same as text block.

[up]Converting objects

You can change the type on object by right-clicking in graph and selecting which type you would prefer to set to this object. Currently available conversions:

From
To
Lose of data
Text block Identifier None
Identifier Text block None

Context menu also has an option “Connect nodes”, which would connect two objects on graph by link. To do this you should select any two nodes and then select “Connect nodes” in context menu or use hotkey C. To delete nodes or links select them and press Delete key.

[up]Drag’n’Drop

SCA’s interaction is mostly based on drag’n’drop technology. To add an object to graph you should hold your mouse on object you want to add and move the mouse to Graph view and when the object appears under your cursor release the mouse button. Graph View recognizes type of everything you drag from SCA’s views automatically, but you can change it. (see converting)

[up]Export graph to image

To export graph to image, select context menu File->Export ..., then select type of image, type in name of file and click "Save".

[up]Filtering objects

Simple

The simplest way to filter objects on graph is to use field under the graph. This field filters object by their path. Filtered objects will always accept red color.

Advanced

To use advanced filter you can call it by context menu Edit->Filter or by clicking on button in the end of simple path-filter line.

SCA uses regexes to recognise if object matches filter. If you don't know regexes you can use our simple choser of type/path/annotation/content for filter. If you know regexes, you can advance filter's regex manually by turning on "Custom RegExp" checkbox. To reset filter click "Reset". "Refresh" button on the right is supposed to refresh regex of filter if it was changed outside of that window. (e.g. in simple filter)

[up]Customizing views.

You can customly place any view in any place on your screen (except Graph view). To do this, hold their titles and drag anywhere you want. You can also stick to each other inside main window. To addition, dragging one view to another merges them to one place and creates tab-switcher on the bottom of that place. To close view click X, to open it or to raise (for tabbed views), use menu Window->Views.
