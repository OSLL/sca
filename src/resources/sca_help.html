Overview

Source Code Analyzer (SCA) is a tool to maintain knowledge base around the sources and projects.

SCA has two main workplaces:

  1. File browser (on the left)
  2. Current view’s tab (on the right), which can be:
    1. Text view
    2. Binary view
    3. Graph view

Workplaces

File browser

You can use file browser as your usual tree-browser: expand folders, fold them etc.

To use file: set current view to view which you want to use with file and double click file. As a default, for Text view it opens file using UTF-8 encoding, but you can force it to open in different encoding by calling context menu on file, selecting “Open in Text viewer as” and then selecting your preferable encoding out of supported ones. File browser supports drag’n’drop.

Current view

Text view

Text view provides viewing files in different encodings: UTF-8, CP-866, ISO-8859-15. It also supports drag’n’drop.

Binary view

Binary view provides viewing files as a HEX-files as any usual hex-editor. On the left you can see current offset of line in the table, in the next column - bytes loaded and separated by spaces, on the right you can see their symbolic representation.

Context menu also has an option “Connect nodes”, which would connect two objects on graph by link. To do this you should select any two nodes and then select “Connect nodes” in context menu or use hotkey C. To delete nodes or links select them and press Delete key.

Drag’n’Drop

SCA’s interaction is mostly based on drag’n’drop technology. To add an object to graph you should hold your mouse on object you want to add and move the mouse to Graph view and when the object appears under your cursor release the mouse button. Graph View recognizes type of everything you drag from SCA’s views automatically, but you can change it. (see Object types)

When you try to drag from text or binary view you will probably have Graph view covered, just drag firstly to it’s tab name without releasing mouse button, then on Graph view itself.

Hotkeys

Where is used
Description
